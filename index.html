CryptR is a new generation Cryptographic authentication protocol based on SHA-3. It is intended to protect against any hacking attempt on the authentication protocol.

CryptR completely eliminates the risk of password exposure by ensuring that all data transmitted over the network do not contain the original password although the authentification depends entirely on the password.

The CryptR Protocol Description

Registration

Step 1: Initialization

The client and the server already have shared a common random number RandomInit (R0) of size 256 bits.
The client chooses his ID and password.
The client generates a random ISC0 of size 1088 bits.
d is a 0-vector of size 256 bits

Step 2: Hashing (two times)

The client uses the SHA3 Keccak hashing function with input R0 concatenated with Password and ISC0.
The first 512 bits of the output is called H'0. This value will not be used and can be ignored.
The last 1088 bits of the output is called FSC'0.
The client then uses the SHA3 Keccak hashing function with input R0 concatenated with d and FSC'0.
The first 512 bits of the output is called H0. This value is sent to the server along with the user ID for registration.

At the end of this step, the user has registered to the server.
The client has stored R0, d and ISC0. The user remembers ID and Password.
The server has stored R0, ID and H0.

Authentication

For the first authentication, n = 0. This value will increment by 1 at each authentication.

Step 3: client-side

The user enters ID and Password in the client.
The client uses the SHA3 Keccak hashing function with input R0 concatenated with Password and ISCn.
The first 512 bits of the output is called H'n. This value will not be used and can be ignored.
The last 1088 bits of the output is called FSC'n.
The client then uses the SHA3 Keccak hashing function with input R0 concatenated with d and FSC'n.
The first 512 bits of the output is called Hn. This is the value previously sent to the server.
The last 1088 bits of the output is called FSCn. This value is sent to the server along with the user ID for authentication.

Step 4: server-side

The server uses the CryptR function with input Hn and FSCn. This will output Rn concatenated with d and FSC'n.
The server compares this Rn with the Rn value that he has stored previously. If the values match, the authentication is successfull. If the values don't match, the authentication failed.
In case of success, the server then calculates the next Rn+1 value for the next authentication. He executes SHA3 with input R0 and FSC'n. Rn+1 is the first 256 bits of the output.
The server sends a ACK message to the client to acknowledge that authentication is successfull.

Step 5: prepare for next authentication

The client calculates the next Rn+1 value for the next authentication. He executes SHA3 with input R0 and FSC'n. Rn+1 is the first 256 bits of the output.
The client then calculates the next Hn+1 value for the next authentication like in step 2.
The client generates a random ISCn+1 of size 1088 bits.
The client uses the SHA3 Keccak hashing function with input Rn+1 concatenated with Password and ISCn+1.
The first 512 bits of the output is H'n+1. This value will not be used and can be ignored.
The last 1088 bits of the output is FSC'n+1.
The client then uses the SHA3 Keccak hashing function with input Rn+1 concatenated with d and FSC'n+1.
The first 512 bits of the output is called Hn+1. This value is sent to the server along with the user ID for the next authentication.


At the end of this step, the client is authenticated to the server and ready for the next authentication.
The client has stored R0, Rn+1, d and ISCn+1. The user remembers ID and Password.
The server has stored R0, Rn+1, ID and Hn+1.
For the next authentication, we repeat from step 3 with n incremented by 1.

The hacking Parameters

RandomInit :

Random number size 256 bits

Password :

weak password concatenated with 0-vector of total size 256 bits

ISC :

Initial Sponge Capacity : random number of size 1088 bits

d :

0-vector of size 256 bits

Function SHA3 :

Keccak hashing function with 512 bits hash output, internal state of size 1600 bits and 24 rounds

H:

Hash: first 512 bits of the SHA3 function output

FSC:

Final Sponge Capacity: last 1088 bits of the SHA3 function output

Function CryptR:

Reverse Keccak hashing function. CryptR takes as input the Hash and the FSC value of a Keccak output, and will output the input of that Keccak function.
